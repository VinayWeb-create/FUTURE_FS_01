UserAuthSense AI Security

An experimental authentication system that combines traditional login methods with behavioral biometrics to detect suspicious or abnormal user activity.

Problem Statement

Traditional authentication systems rely solely on credentials like username and password—a single point of failure.

If credentials are compromised through phishing, data breaches, or social engineering, systems often fail to detect unauthorized access until damage is already done, as they cannot distinguish between legitimate users and attackers with stolen credentials.

Solution Approach

UserAuthSense adds an additional behavioral layer to authentication that works alongside traditional credentials.

Instead of trusting credentials alone, the system analyzes how a user behaves during login—creating a behavioral fingerprint that's much harder to replicate than a password.

System Architecture

The system captures behavioral data during login attempts, including typing rhythm, mouse movement patterns, and interaction timing.

This data is processed using a backend ML model that compares it against the user's historical behavioral profile and assigns a risk score to each session. High-risk sessions can trigger additional verification, be flagged for review, or denied access entirely.

Tech Stack & Design

Frontend: HTML, CSS, JavaScript—event-based behavioral data capture without impacting UX

Backend: Python & Flask—request handling, ML inference, and risk evaluation

ML Models: Behavioral pattern analysis using anomaly detection algorithms

Security: Session management with risk-based access control and adaptive authentication

Key Features

  • Passive behavioral data capture during authentication
  • Real-time risk scoring for each login attempt
  • Detection of abnormal interaction patterns and timing
  • Session-level security decisions with confidence levels
  • Adaptive thresholds that learn from user behavior over time
  • Privacy-focused design with minimal data retention

Challenges & Solutions

Challenge: Identifying useful behavioral signals that distinguish users

Solution: Focused on relative behavior patterns and temporal features instead of raw values

Challenge: Handling noisy and inconsistent user behavioral data

Solution: Added normalization, outlier detection, and threshold-based filtering

Challenge: Avoiding false positives that frustrate genuine users

Solution: Designed the system to flag risk rather than hard-block, with graceful degradation

Challenge: Balancing security enhancement with user experience

Solution: Kept behavioral capture invisible to users—no additional friction in the login flow

Security Philosophy

Security systems must assume credentials can be compromised. Passwords can be guessed, stolen, or phished—but behavioral patterns are much harder to replicate.

This project deepened my understanding of multi-layered security, the importance of defense in depth, authentication flows, anomaly detection techniques, and the practical trade-offs between security and usability in ML-powered systems.

Future Enhancements
